# Roles & who does what

Campfire keeps **running the server** separate from **running a game**. A full
explanation lives in [Admin vs DM](../administration/access-model.md); the short
version:

- **Server admin** — operates the server (accounts, settings, rule packs). Need
  not be anyone's DM.
- **DM** — runs a campaign. Anyone who creates a campaign is its DM; you don't
  need to be a server admin.
- **Player** — owns a character, tracks objectives, keeps notes, rolls dice.
- **Viewer** — read-only, plus quick notes to the DM.

Server role (`admin`/`user`) and campaign role (`dm`/`player`/`viewer`) are
independent — one account can be a plain user on the server and the DM of its own
table. See [Admin vs DM](../administration/access-model.md) for how players get
accounts (admin-created, DM invites, or self-service signup).
